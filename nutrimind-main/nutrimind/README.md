# 🥗 NutriMind Flutter App — Firebase Edition

**Your Personal Editorial Food Sanctuary** for Davao, Philippines.
Full-stack Flutter app with Firebase Authentication, Firestore database, and fully functional UI.

---

## 🔥 Firebase Setup (Required Before Running)

### Step 1 — Create Firebase Project
1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Click **Add project** → Name it `nutrimind`
3. Enable **Google Analytics** (optional)

### Step 2 — Enable Authentication
1. Firebase Console → **Authentication** → **Sign-in method**
2. Enable **Email/Password**
3. Enable **Google** (add your SHA-1 key for Android)

### Step 3 — Create Firestore Database
1. Firebase Console → **Firestore Database** → **Create database**
2. Start in **test mode** (change rules before production)
3. Choose your region (e.g. `asia-southeast1` for Philippines)

### Step 4 — Firestore Security Rules
Go to **Firestore → Rules** and paste:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /users/{userId}/meals/{mealId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /posts/{postId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update: if request.auth != null;
      allow delete: if request.auth != null && resource.data.userId == request.auth.uid;
    }
    match /posts/{postId}/comments/{commentId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
    }
  }
}
```

### Step 5 — Connect Flutter to Firebase

Install the FlutterFire CLI:
```bash
dart pub global activate flutterfire_cli
```

Run in your project root:
```bash
flutterfire configure
```
Select your Firebase project. This auto-generates `lib/firebase_options.dart` with correct values.

### Step 6 — Android Setup
In `android/app/build.gradle`, make sure:
```gradle
defaultConfig {
    minSdkVersion 21
    ...
}
```

For Google Sign-In on Android, add your SHA-1 key:
```bash
cd android && ./gradlew signingReport
```
Copy the **SHA-1** from `debug` variant → paste in Firebase Console → Project Settings → Your Android app.

### Step 7 — iOS Setup (if needed)
In `ios/Runner/Info.plist`, add the `REVERSED_CLIENT_ID` from your `GoogleService-Info.plist`:
```xml
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>YOUR_REVERSED_CLIENT_ID</string>
    </array>
  </dict>
</array>
```

## 🗄️ Firestore Data Structure

```
users/{uid}
  name, email, photoUrl, location
  goal, gender, height, weight, age
  dailyBudget, allowNonLocal, budgetBuffer
  isPremium, createdAt

users/{uid}/meals/{mealId}
  name, type, price, calories
  status (upcoming/ready/logged)
  date, loggedAt
  protein, carbs, fat, ingredients

posts/{postId}
  userId, userName, userPhotoUrl
  content, category, tags
  likes[], commentCount
  imageUrl, createdAt

posts/{postId}/comments/{commentId}
  userId, userName, content, createdAt
```
